INTRODUCTION                                                *tsgo-introduction*

tsgo.nvim adds a VS Code-like editor-experience layer around the native
TypeScript tsgo LSP server. It does not wrap tsgo or translate protocols. The
server already speaks LSP.

The main feature is import hygiene: a single command that adds missing imports,
removes unused imports, and organizes imports.

It also exposes completion polish helpers for completion engines such as
blink.cmp, runtime inspection, and health checks.

==============================================================================
SETUP                                                            *tsgo-setup*

>
  require("tsgo").setup({
    inlay_hints = { auto_enable = true },
    keymaps = {
      enable = true,
      imports_format = "<leader>iI",
    },
  })
<

By default, setup registers the tsgo LSP config when |vim.lsp.config()| is
available:
>
  cmd = { "tsgo", "--lsp", "-stdio" }
<

If another plugin owns LSP setup:
>
  require("tsgo").setup({
    setup_lsp = false,
    keymaps = { enable = true },
  })
<

COMMANDS                                                      *tsgo-commands*

                                                        *:TsgoOrganizeImports*
:TsgoOrganizeImports
    Run the source.organizeImports LSP code action.

                                                        *:TsgoAddMissingImports*
:TsgoAddMissingImports
    Run the source.addMissingImports.ts LSP code action.

                                                        *:TsgoRemoveUnused*
:TsgoRemoveUnused
    Run the source.removeUnused.ts LSP code action.

                                                        *:TsgoFixAll*
:TsgoFixAll
    Run the source.fixAll.ts LSP code action.

                                                        *:TsgoImports*
:TsgoImports
    Add missing imports, remove unused imports, and organize imports.

:TsgoImports!
    Run import hygiene and format afterward.

Set keymaps.imports_format to bind the same workflow as |:TsgoImports!|.

                                                        *:TsgoSourceDefinition*
:TsgoSourceDefinition
    Try TypeScript source definition and fall back to standard LSP definition.

                                                        *:TsgoSignatureHelp*
:TsgoSignatureHelp
    Request tsgo signature help with VS Code-like context so documentation-rich
    signatures are shown when the server returns them.

                                                        *:TsgoToggleInlayHints*
:TsgoToggleInlayHints
    Toggle Neovim's inlay hint renderer for the current buffer.

                                                        *:TsgoInfo*
:TsgoInfo
    Show the resolved tsgo command, executable availability, and active client
    count.

==============================================================================
INLAY HINTS                                                *tsgo-inlay-hints*

tsgo advertises inlay hints, but the TypeScript hint categories need to be
enabled through settings. tsgo.nvim enables a quiet default for both JavaScript
and TypeScript: parameter-name hints for all arguments and enum member value
hints. Variable, parameter type, property type, and function return hints are
off by default because they tend to add too much visual noise.

Override the settings if you want more type hints:
>
  require("tsgo").setup({
    settings = {
      typescript = {
        inlayHints = {
          parameterTypes = { enabled = true },
          variableTypes = { enabled = true },
          functionLikeReturnTypes = { enabled = true },
        },
      },
    },
  })
<

==============================================================================
LUA API                                                           *tsgo-api*

>
  require("tsgo").actions.organize_imports()
  require("tsgo").actions.add_missing_imports()
  require("tsgo").actions.remove_unused()
  require("tsgo").actions.fix_all()
  require("tsgo").actions.imports({ format = true })
  require("tsgo").actions.source_definition()
  require("tsgo").inlay_hints.toggle()
  require("tsgo").signature.help()
  require("tsgo").compat.blink()
<

Use the blink.cmp compatibility patch by merging it into your blink config:
>
  local tsgo_blink = require("tsgo").compat.blink()
  require("blink.cmp").setup(vim.tbl_deep_extend("force", {
    -- your blink config
  }, tsgo_blink))
<

By default, the blink patch buckets typed member completions by prefix match, so
`.len` ranks `length` before fuzzy typo matches like `entries`.
Blank member completion after just `.` still preserves TypeScript's LSP
sortText ordering.

Disable that sorter if you want pure blink/LSP ordering:
>
  local tsgo_blink = require("tsgo").compat.blink({
    completion = {
      member_prefix_sort = false,
    },
  })
<

Run health checks with:
>
  :checkhealth tsgo
<
